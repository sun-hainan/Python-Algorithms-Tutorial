## 第一部分：什么是梯度？

### 直观理解

```
梯度 = 函数在某一点上升最快的方向

下山问题：
- 你在山上某点，想下山到山谷
- 梯度告诉你往哪个方向走最陡
- 负梯度方向 = 下山最快的方向
```

### 数学定义

```
对于函数 f(x, y)：
梯度 = (∂f/∂x, ∂f/∂y)

它是一个向量，指向函数值增加最快的方向
```

### 例子

```python
import numpy as np

def func(x, y):
    """二元函数"""
    return x**2 + y**2

def gradient(x, y):
    """梯度 ∂f/∂x = 2x, ∂f/∂y = 2y"""
    return np.array([2*x, 2*y])

# 在点 (3, 4) 处
x, y = 3, 4
g = gradient(x, y)
print(f"函数值: f(3,4) = {func(x, y)}")
print(f"梯度: ({g[0]}, {g[1]})")
print(f"梯度方向: 往 ({g[0]}, {g[1]}) 走，函数值增加最快")
print(f"负梯度方向: 往 ({-g[0]}, {-g[1]}) 走，函数值减小最快")
```

## 第二部分：梯度下降算法

### 更新规则

```
θ_new = θ_old - learning_rate × ∇L(θ)

其中：
- θ: 模型参数
- ∇L(θ): 损失函数的梯度
- learning_rate: 学习率（步长）
```

### 形象理解

```
想象你在山上每走一步：
- 看一眼脚下最陡的方向（计算梯度）
- 决定步子迈多大（学习率）
- 迈出一步（更新参数）
- 重复直到山谷底部（收敛）
```

## 第三部分：三种梯度下降

### 1. 批量梯度下降（BGD）

```python
def batch_gradient_descent(X, y, lr=0.01, epochs=1000):
    """
    批量梯度下降

    每一步使用所有样本计算梯度
    """
    n_samples, n_features = X.shape
    w = np.zeros(n_features)

    for epoch in range(epochs):
        # 使用所有样本计算梯度
        y_pred = np.dot(X, w)
        error = y_pred - y
        gradient = np.dot(X.T, error) / n_samples

        w -= lr * gradient

        if epoch % 100 == 0:
            loss = np.mean(error ** 2)
            print(f"Epoch {epoch}: loss = {loss:.4f}")

    return w
```

### 2. 随机梯度下降（SGD）

```python
def sgd(X, y, lr=0.01, epochs=1000, batch_size=1):
    """
    随机梯度下降

    每一步只用一个样本计算梯度
    """
    n_samples = len(y)

    w = np.zeros(X.shape[1])

    for epoch in range(epochs):
        # 打乱数据
        indices = np.random.permutation(n_samples)

        for i in indices:
            xi = X[i:i+1]
            yi = y[i:i+1]

            # 单个样本的梯度
            error = np.dot(xi, w) - yi
            gradient = xi.T * error

            w -= lr * gradient.flatten()

    return w
```

### 3. 小批量梯度下降（Mini-Batch GD）

```python
def mini_batch_gd(X, y, lr=0.01, epochs=1000, batch_size=32):
    """
    小批量梯度下降

    每一步使用 batch_size 个样本
    """
    n_samples = len(y)

    w = np.zeros(X.shape[1])

    for epoch in range(epochs):
        indices = np.random.permutation(n_samples)

        for start in range(0, n_samples, batch_size):
            end = min(start + batch_size, n_samples)
            X_batch = X[indices[start:end]]
            y_batch = y[indices[start:end]]

            # 小批量样本的梯度
            y_pred = np.dot(X_batch, w)
            error = y_pred - y_batch
            gradient = np.dot(X_batch.T, error) / len(y_batch)

            w -= lr * gradient

    return w
```

### 对比

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| BGD | 稳定收敛 | 慢、大数据难处理 | 小数据集 |
| SGD | 快、在线学习 | 震荡、不稳定 | 大数据集 |
| Mini-Batch | 平衡速度与稳定 | 需要调batch_size | **最常用** |

## 第四部分：学习率的影响

```python
"""
学习率太大会怎样？

lr = 1.5（过大）
→ 参数在最优解附近振荡
→ 甚至发散！

lr = 0.001（过小）
→ 收敛太慢
→ 可能陷入局部最优
```

### 学习率调度

```python
def learning_rate_schedule(epoch, initial_lr=0.1, decay_rate=0.95):
    """学习率衰减"""
    return initial_lr * (decay_rate ** epoch)


# 常用策略
schedules = {
    # 阶梯衰减
    'step': lambda e: 0.1 * (0.1 ** (e // 30)),

    # 余弦退火
    'cosine': lambda e: 0.5 * (1 + np.cos(np.pi * e / epochs)) * initial_lr,

    # 指数衰减
    'exponential': lambda e: initial_lr * (0.95 ** e),

    # Warmup + Decay
    'warmup': lambda e: initial_lr * min(1.0, e / 10) if e < 10 else initial_lr * (0.1 ** (e // 50))
}
```

## 第五部分：动量（Momentum）

### 物理直觉

```
普通梯度下降：
- 每一步只看当前梯度
- 容易在沟壑中震荡

动量：
- 积累之前的方向
- 类似滚下山坡的惯性
- 能加速收敛、减少震荡
```

```python
def gradient_descent_momentum(X, y, lr=0.01, epochs=1000, momentum=0.9):
    """带动量的梯度下降"""
    n_samples, n_features = X.shape
    w = np.zeros(n_features)

    v_w = np.zeros(n_features)  # 速度

    for epoch in range(epochs):
        y_pred = np.dot(X, w)
        error = y_pred - y
        gradient = np.dot(X.T, error) / n_samples

        # 动量更新
        v_w = momentum * v_w - lr * gradient
        w += v_w

    return w
```

## 第六部分：Adam优化器

```python
def adam(X, y, lr=0.001, epochs=1000, beta1=0.9, beta2=0.999, eps=1e-8):
    """
    Adam: Adaptive Moment Estimation

    结合了动量和RMSProp的优点
    """
    n_samples, n_features = X.shape
    w = np.zeros(n_features)

    # 一阶矩估计（类似动量）
    m_w = np.zeros(n_features)
    # 二阶矩估计（类似RMSProp）
    v_w = np.zeros(n_features)

    for epoch in range(epochs):
        y_pred = np.dot(X, w)
        error = y_pred - y
        gradient = np.dot(X.T, error) / n_samples

        # 更新一阶、二阶矩
        m_w = beta1 * m_w + (1 - beta1) * gradient
        v_w = beta2 * v_w + (1 - beta2) * (gradient ** 2)

        # 偏差校正
        m_hat = m_w / (1 - beta1 ** (epoch + 1))
        v_hat = v_w / (1 - beta2 ** (epoch + 1))

        # 更新参数
        w -= lr * m_hat / (np.sqrt(v_hat) + eps)

    return w
```

### 为什么Adam常用？

```
1. 自适应学习率：每个参数有自己的学习率
2. 动量累积：加速收敛
3. 对超参数不敏感：默认参数通常效果不错
4. 适合大多数场景
```

## ✅ 小结

1. **梯度下降**通过沿负梯度方向更新参数来最小化损失
2. 三种变体：批量、SGD、小批量（最常用）
3. **学习率**是关键超参数，需要仔细调优
4. 动量累积能加速收敛、减少震荡
5. **Adam**是最常用的优化器，自适应学习率
