## 第一部分：什么是排序？

**生活中的排序：**
- 考试完排队，按身高从矮到高
- 字典里的单词，按字母顺序排列
- 书架上的书，按书名从 A 到 Z 排列

把一堆数字，按照从小到大或从大到小的顺序重新排列。

```
排序前: [64, 34, 25, 12, 22]
排序后: [12, 22, 25, 34, 64]
```

## 第二部分：快速排序的核心思想

### 🍕 切披萨的故事

想象你有一个超级大披萨，上面有各种配料：

```
[蘑菇, 火腿, 菠萝, 虾, 橄榄, 青椒, 玉米, 香肠]
  0     1     2    3     4     5    6     7
```

**目标：** 让左边的都比右边的小。

### 怎么切？

**第1步：选一个"基准"披萨**

随便选一个配料，比如"菠萝"（第2个位置）。

**第2步：把小的放左边，大的放右边**

把"菠萝"拿出来：
```
左边: [蘑菇, 火腿]      菠萝(拿出来)      右边: [虾, 橄榄, 青椒, 玉米, 香肠]
```

处理后：
```
[蘑菇, 火腿, 菠萝, 虾, 橄榄, 青椒, 玉米, 香肠]
        ↑
     菠萝在中间，左边的小，右边的大
```

**第3步：基准位置固定了！**

左边的都是比菠萝小的，右边的都是比菠萝大的。
以后不管怎么排序，菠萝都不会跑到别的地方去了。

**第4步：对左右两边重复这个过程**

对左边 `[蘑菇, 火腿]` 做同样的事情
对右边 `[虾, 橄榄, 青椒, 玉米, 香肠]` 做同样的事情

### 快速排序的核心

```
1. 选一个基准
2. 把小的放左边，大的放右边
3. 基准位置固定了
4. 对左右两边重复这个过程
```

## 第三部分：代码逐行解释

```python
# ============================================================
# 函数名：quick_sort
# 参数：arr（一组需要排序的数字）
# 返回值：排好序的新数组
# ============================================================
def quick_sort(arr):

    # -------- 递归终止条件 --------
    # 如果数组只有0个或1个元素，直接返回（已经是排好序的了）
    if len(arr) <= 1:
        return arr

    # -------- 选择基准 --------
    # 选中间位置的数字作为"基准"（pivot）
    # 例如：[1,2,3,4,5] 的中间位置是 2
    mid = len(arr) // 2      # // 是整除符号，5//2=2，7//2=3
    pivot = arr[mid]          # 取出中间位置的那个数作为基准

    # -------- 准备分组容器 --------
    left = []     # 用来存放所有小于等于基准的数字
    right = []    # 用来存放所有大于基准的数字

    # -------- 遍历数组，根据大小放入对应容器 --------
    for num in arr:

        # 如果当前数字就是基准，跳过它（基准不参与比较）
        if num == arr[mid]:
            continue        # continue 表示跳过后面的代码，继续下一次循环

        # -------- 比较并分组 --------
        if num <= pivot:
            left.append(num)       # append = 追加到列表末尾
        else:
            right.append(num)

    # -------- 递归排序左右两边，然后合并 --------
    sorted_left = quick_sort(left)   # 递归：对"左边"继续排序
    sorted_right = quick_sort(right) # 递归：对"右边"继续排序

    # -------- 合并三部分 --------
    return sorted_left + [pivot] + sorted_right
```

## 第四部分：实际运行代码

```python
def quick_sort(arr):
    """
    快速排序 - 分治策略（Divide and Conquer）

    参数:
        arr: 待排序的数字列表
    返回值:
        排好序的新列表
    """
    # -------- 递归终止条件 --------
    if len(arr) <= 1:
        return arr

    # -------- 选择基准 --------
    mid = len(arr) // 2
    pivot = arr[mid]

    # -------- 分组 --------
    left = []
    right = []
    for num in arr:
        if num == arr[mid]:
            continue
        if num <= pivot:
            left.append(num)
        else:
            right.append(num)

    # -------- 递归 + 合并 --------
    return quick_sort(left) + [pivot] + quick_sort(right)


# ========================== 测试 ==========================

print("测试1：简单数字排序")
test_arr = [64, 34, 25, 12, 22, 11, 90]
print(f"  排序前: {test_arr}")
print(f"  排序后: {quick_sort(test_arr)}")
# 输出: [11, 12, 22, 25, 34, 64, 90]

print("\n测试2：更复杂的数组")
print(f"  排序后: {quick_sort([5, 2, 9, 1, 7, 6, 3])}")
# 输出: [1, 2, 3, 5, 6, 7, 9]

print("\n测试3：已有顺序的数组")
print(f"  排序后: {quick_sort([1, 2, 3, 4, 5])}")
# 输出: [1, 2, 3, 4, 5]

print("\n测试4：倒序的数组")
print(f"  排序后: {quick_sort([5, 4, 3, 2, 1])}")
# 输出: [1, 2, 3, 4, 5]
```

## 第六部分：排序算法对比

### 时间复杂度对比

| 算法 | 时间复杂度 | 适合场景 |
|------|-----------|----------|
| 冒泡排序 | O(n²) | 教学/数据量小 |
| **快速排序** ⭐ | **O(n log n)** | **通用首选** |
| 归并排序 | O(n log n) | 需要稳定排序 |
| 堆排序 | O(n log n) | Top-K 问题 |

### 为什么快速排序是通用的？

```
100个数字时：
- 冒泡：10000次比较
- 快速排序：660次比较
快了近15倍！
```

## 第七部分：名词解释

### 递归
**定义：** 函数调用自己。

```python
def count_down(n):
    print(n)
    if n > 1:
        count_down(n - 1)  # 调用自己

count_down(5)  # 输出：5, 4, 3, 2, 1
```

### 基准 (Pivot)
**定义：** 参照物。比基准小的放左边，比基准大的放右边。

### 时间复杂度
**定义：** 算法运行时间与数据量的关系。
- O(n²)：数据量×数据量
- O(n log n)：数据量×log(数据量)

### 稳定性
**定义：** 排序后，相等元素的相对位置是否保持不变。
- 稳定：位置不变
- 不稳定：位置可能变

> 快速排序是不稳定的。

### 原地排序
**定义：** 不需要额外申请新的内存空间。

## ✅ 小结

1. **快速排序**使用分治策略
2. 选基准 → 分左右 → 递归 → 合并
3. 平均时间复杂度 O(n log n)
4. 是最常用的通用排序算法
