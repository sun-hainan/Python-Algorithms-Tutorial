## 第一部分：什么是归并排序？

### 合并的故事

想象有两叠已经按顺序排好的扑克牌：

```
左叠：[3, 5, 7]      右叠：[1, 4, 6]
```

怎么把它们合并成一叠有序的？

**方法：**
1. 比较两叠的顶部
2. 把较小的放到结果堆
3. 重复直到完成

```
Step 1: 比较 3 和 1，取 1  → 结果：[1]
Step 2: 比较 3 和 4，取 3  → 结果：[1, 3]
Step 3: 比较 5 和 4，取 4  → 结果：[1, 3, 4]
Step 4: 比较 5 和 6，取 5  → 结果：[1, 3, 4, 5]
Step 5: 比较 7 和 6，取 6  → 结果：[1, 3, 4, 5, 6]
Step 6: 左边剩余，取 7    → 结果：[1, 3, 4, 5, 6, 7]
```

### 归并排序的核心

**分治策略（Divide and Conquer）：**
1. **分解：** 把数组从中间分成两半
2. **解决：** 递归地对左右两部分排序
3. **合并：** 合并两个有序数组

```
[38, 27, 43, 3, 9, 82, 10]
     ↓ 分解
[38, 27, 43, 3]  [9, 82, 10]
     ↓ 分解
[38, 27]  [43, 3]  [9, 82]  [10]
     ↓ 分解
[38] [27] [43] [3] [9] [82] [10]
     ↓ 合并
[27, 38]  [3, 43]  [9, 82]  [10]
     ↓ 合并
[3, 27, 38, 43]  [9, 10, 82]
     ↓ 合并
[3, 9, 10, 27, 38, 43, 82]
```

## 第二部分：图解合并过程

### 合并两个有序数组

```
左数组：[1, 3, 5]      右数组：[2, 4, 6]
            ↓
比较：1 vs 2 → 取 1      结果：[1]
比较：3 vs 2 → 取 2      结果：[1, 2]
比较：3 vs 4 → 取 3      结果：[1, 2, 3]
比较：5 vs 4 → 取 4      结果：[1, 2, 3, 4]
比较：5 vs 6 → 取 5      结果：[1, 2, 3, 4, 5]
左数组空，取右数组剩余 → 结果：[1, 2, 3, 4, 5, 6]
```

## 第三部分：代码逐行解释

```python
def merge_sort(arr):
    """
    归并排序 - 分治策略

    参数:
        arr: 待排序的数组
    返回值:
        排序后的新数组
    """

    # -------- 递归终止条件 --------
    # 当数组为空或只有一个元素时，直接返回
    if len(arr) <= 1:
        return arr

    # -------- 第1步：分解 --------
    # 从中间把数组分成两半
    mid = len(arr) // 2
    left_half = arr[:mid]    # 左半部分
    right_half = arr[mid:]   # 右半部分

    # -------- 第2步：递归排序 --------
    # 对左半部分递归调用归并排序
    sorted_left = merge_sort(left_half)
    # 对右半部分递归调用归并排序
    sorted_right = merge_sort(right_half)

    # -------- 第3步：合并 --------
    # 合并两个有序数组
    return merge(sorted_left, sorted_right)


def merge(left, right):
    """
    合并两个有序数组

    参数:
        left: 左边的有序数组
        right: 右边的有序数组
    返回值:
        合并后的有序数组
    """

    # -------- 初始化 --------
    result = []    # 存放结果的列表
    i = j = 0     # 两个指针，分别指向 left 和 right

    # -------- 比较并合并 --------
    # 同时遍历两个数组，把较小的放入结果
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            # left[i] 较小或相等，放入结果
            result.append(left[i])
            i += 1
        else:
            # right[j] 较小，放入结果
            result.append(right[j])
            j += 1

    # -------- 处理剩余元素 --------
    # 把剩余的元素直接添加到结果
    # 最多只有一个数组有剩余
    result.extend(left[i:])  # 添加 left 剩余的
    result.extend(right[j:]) # 添加 right 剩余的

    return result
```

## 第四部分：实际运行代码

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

## 第六部分：排序算法对比

### 归并排序 vs 快速排序

| 特点 | 归并排序 | 快速排序 |
|------|----------|----------|
| 时间复杂度 | O(n log n) | O(n log n) |
| 空间复杂度 | O(n) | O(log n) |
| 稳定性 | **稳定** ⭐ | 不稳定 |
| 实现方式 | 自底向上天然适合递归 | 基准选择影响大 |

### 何时使用归并排序？

✅ **适合使用：**
- 需要稳定排序时 ⭐
- 大数据外部排序
- 链表排序
- 数据完全无序

## 第七部分：名词解释

### 分治策略

**定义：** 把大问题分解成小问题，分别解决，再合并结果。

```
大问题：排序整个数组
    ↓ 分解
小问题1：排序左半部分
小问题2：排序右半部分
    ↓ 合并
大问题结果：整个数组有序
```

### 稳定性

**定义：** 排序后，相等元素的相对位置是否保持不变。

```python
# 稳定排序的例子
原始：[2₁, 1, 2₂]
排序后：[1, 2₁, 2₂]  ← 2₁ 和 2₂ 的顺序保持不变

# 不稳定排序的例子
原始：[2₁, 1, 2₂]
排序后：[1, 2₂, 2₁]  ← 顺序变了
```

### 递归

**定义：** 函数调用自己来解决问题。

```python
def merge_sort(arr):
    if len(arr) <= 1:  # 终止条件
        return arr
    # ... 分解 ...
    return merge(merge_sort(left), merge_sort(right))  # 递归调用
```

### 双指针

**定义：** 用两个指针同时遍历两个数组。

```python
i = 0  # 指向 left
j = 0  # 指向 right
while i < len(left) and j < len(right):
    # 比较两个指针位置的元素
    if left[i] <= right[j]:
        result.append(left[i])
        i += 1
    else:
        result.append(right[j])
        j += 1
```
