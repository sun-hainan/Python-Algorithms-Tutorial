# 线性回归

> _从一条最简单的直线开始，走进机器学习的世界_

---

## 🌟 先看一个生活中的例子

### 例子：猜猜这套房子多少钱？

假设你现在是一个房产中介，经理给你一个任务：

> **有一套 120 平米的房子，你能猜猜它大概能卖多少钱吗？**

你会怎么猜？

你可能会想：
- 我之前卖过一套 100 平米的房子，卖了 **300 万**
- 面积越大，价格越高，这是常识
- 每平米大概 3 万块钱
- 那 120 平米的房子，大概就是 **3 万 × 120 = 360 万** 左右

恭喜你！**你刚才做的，就是线性回归！**

---

## 🎯 什么是线性回归？

### 线性回归要做的事情

**线性回归**就是：**找到一条直线，让它最好地代表一堆数据点的规律，然后用这条直线去预测新数据。**

就像这样：

```
房价（万）
  500 │        ╱
      │       ╱  ← 拟合的直线
  400 │      ╱
      │     ╱
  300 │    ╱  ●  (100㎡, 300万)
      │   ╱
  200 │  ╱
      │ ╱
      └───────────────── 面积（平米）
        60   80  100  120  140
```

### 线性回归的名字解读

- **"线性"**：数据画在图上，像一条直线
- **"回归"**：预测一个**连续**的值（比如 360 万、372 万），而不是分类（猫或狗）

---

## 📐 第一步：建立数学模型

### 只有一个特征的情况（一元线性回归）

假设我们只有**面积**这一个特征来预测房价：

```
房价 = w × 面积 + b
```

就这么简单！

其中：
- **w** 叫做**斜率**（slope），意思是每增加 1 平米，房价增加多少
- **b** 叫做**截距**（intercept），意思是即使 0 平米的房子也有个基础价格
- **x** 是面积
- **y** 是预测的房价

### 举个例子

假设 w = 3，b = 20：

```
房价 = 3 × 面积 + 20

如果面积 = 100平米：
房价 = 3 × 100 + 20 = 320 万

如果面积 = 120平米：
房价 = 3 × 120 + 20 = 380 万

如果面积 = 80平米：
房价 = 3 × 80 + 20 = 260 万
```

### 有多个特征的情况（多元线性回归）

现实中，预测房价不会只看面积，还有：
- 房间数量
- 距离地铁多远
- 楼层高低
- 建造年份
- 是否有电梯

这时候公式就变成：

```
房价 = w₁×面积 + w₂×房间数 + w₃×地铁距离 + ... + b
```

为了写起来方便，我们把 b 也融进公式里：

```
房价 = w₀×1 + w₁×面积 + w₂×房间数 + w₃×地铁距离 + ...
```

## 🤔 第二步：怎么知道 w 和 b 好不好？

### 先看一个具体的例子

假设我们知道 5 套房子的真实价格：

| 面积（平米） | 真实房价（万） |
|-------------|---------------|
| 80 | 280 |
| 100 | 340 |
| 120 | 400 |
| 140 | 450 |
| 160 | 520 |

### 用线性模型预测一下

假设我们猜 w = 3，b = 0（随便猜的）：

| 面积 | 真实房价 | 预测房价 | 误差 |
|------|---------|---------|------|
| 80 | 280 | 3×80 = 240 | 40 |
| 100 | 340 | 3×100 = 300 | 40 |
| 120 | 400 | 3×120 = 360 | 40 |
| 140 | 450 | 3×140 = 420 | 30 |
| 160 | 520 | 3×160 = 480 | 40 |

**误差** = 真实值 - 预测值

### 怎么衡量整体误差？

误差有正有负（有的预测高了，有的预测低了），直接相加会互相抵消。

所以我们用**平方误差**：

```
误差² = (真实值 - 预测值)²
```

| 面积 | 真实房价 | 预测房价 | 误差 | 误差² |
|------|---------|---------|------|-------|
| 80 | 280 | 240 | 40 | 1,600 |
| 100 | 340 | 300 | 40 | 1,600 |
| 120 | 400 | 360 | 40 | 1,600 |
| 140 | 450 | 420 | 30 | 900 |
| 160 | 520 | 480 | 40 | 1,600 |

**总误差²** = 1600 + 1600 + 1600 + 900 + 1600 = **7,300**

**平均误差²** = 7300 / 5 = **1,460**

这就是 **MSE（均方误差）**！

### 损失函数（Loss Function）

**我们的目标是：找到最合适的 w 和 b，让这个误差最小！**

损失函数就是：

```
L(w, b) = Σᵢ (yᵢ - (w·xᵢ + b))² / n

意思是：每个样本的误差平方加起来，再除以样本数量
```

---

## 🧮 第三步：怎么找到最好的 w 和 b？

有两种方法：
1. **最小二乘法**：直接代数计算，一步到位
2. **梯度下降法**：慢慢迭代，一步一步逼近最优解

---

### 方法一：最小二乘法（适合初学者理解）

#### 核心思想

我们要求的 w 和 b，要让**所有误差平方的和最小**。

#### 公式推导（一步一步来）

我们有：
```
(x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ)  共n个样本
```

目标是 minimize: `L = Σ(yᵢ - wxᵢ - b)²`

**第一步：对 b 求导**

```
∂L/∂b = Σ 2(yᵢ - wxᵢ - b) × (-1) = 0

展开：
-2Σ(yᵢ - wxᵢ - b) = 0
Σyᵢ - wΣxᵢ - nb = 0
nb = Σyᵢ - wΣxᵢ
b = (Σyᵢ - wΣxᵢ) / n
b = ȳ - wx̄
```

其中 `ȳ` 是 y 的平均值，`x̄` 是 x 的平均值。

**第二步：对 w 求导**

```
∂L/∂w = Σ 2(yᵢ - wxᵢ - b) × (-xᵢ) = 0

展开：
-2Σxᵢ(yᵢ - wxᵢ - b) = 0
Σxᵢyᵢ - wΣxᵢ² - bΣxᵢ = 0
```

代入 b = ȳ - wx̄：

```
Σxᵢyᵢ - wΣxᵢ² - (ȳ - wx̄)Σxᵢ = 0
Σxᵢyᵢ - wΣxᵢ² - ȳΣxᵢ + wx̄Σxᵢ = 0
```

移项：

```
w(Σxᵢ² - x̄Σxᵢ) = Σxᵢyᵢ - ȳΣxᵢ
```

所以：

```
w = (Σxᵢyᵢ - ȳΣxᵢ) / (Σxᵢ² - x̄Σxᵢ)
```

或者写成更常见的形式：

```
w = Σ(xᵢ - x̄)(yᵢ - ȳ) / Σ(xᵢ - x̄)²
```

#### 最终公式

```
w = Σ(xᵢ - x̄)(yᵢ - ȳ) / Σ(xᵢ - x̄)²
b = ȳ - wx̄
```

#### 手算一遍

数据：
| i | xᵢ | yᵢ |
|---|-----|-----|
| 1 | 80 | 280 |
| 2 | 100 | 340 |
| 3 | 120 | 400 |
| 4 | 140 | 450 |
| 5 | 160 | 520 |

**第一步：算平均值**

```
x̄ = (80 + 100 + 120 + 140 + 160) / 5 = 600 / 5 = 120
ȳ = (280 + 340 + 400 + 450 + 520) / 5 = 1990 / 5 = 398
```

**第二步：算每个 (xᵢ - x̄)(yᵢ - ȳ)**

| i | xᵢ - x̄ | yᵢ - ȳ | 乘积 |
|---|---------|---------|------|
| 1 | 80-120 = -40 | 280-398 = -118 | (-40)×(-118) = 4,720 |
| 2 | 100-120 = -20 | 340-398 = -58 | (-20)×(-58) = 1,160 |
| 3 | 120-120 = 0 | 400-398 = 2 | 0×2 = 0 |
| 4 | 140-120 = 20 | 450-398 = 52 | 20×52 = 1,040 |
| 5 | 160-120 = 40 | 520-398 = 122 | 40×122 = 4,880 |
| **合计** | | | **11,800** |

**第三步：算每个 (xᵢ - x̄)²**

| i | xᵢ - x̄ | (xᵢ - x̄)² |
|---|---------|-----------|
| 1 | -40 | 1,600 |
| 2 | -20 | 400 |
| 3 | 0 | 0 |
| 4 | 20 | 400 |
| 5 | 40 | 1,600 |
| **合计** | | **4,000** |

**第四步：算 w 和 b**

```
w = 11,800 / 4,000 = 2.95
b = 398 - 2.95 × 120 = 398 - 354 = 44
```

**所以最终公式是：**

```
房价 = 2.95 × 面积 + 44
```

验证一下：
- 80平米：2.95×80 + 44 = 236 + 44 = 280 ✓
- 100平米：2.95×100 + 44 = 295 + 44 = 339 ≈ 340 ✓
- 120平米：2.95×120 + 44 = 354 + 44 = 398 ≈ 400 ✓

完美！

### 方法二：梯度下降法（更重要，后面深度学习用得多）

#### 直观理解

想象你在一座大山的某个位置，天黑了，你看不见路，但你知道山脚下有个小镇。你想下山，怎么走？

你可能会：
1. 低头看看脚下，感受一下哪个方向是下坡
2. 迈一小步，往下坡的方向走
3. 停下来，再感受一下
4. 再走一步
5. 重复...直到你觉得已经到了山脚

**梯度下降就是这个思路！**

#### 什么是梯度？

**梯度**就是山坡上最陡的上坡方向。

我们要往**下**走，所以要走**负梯度**方向。

#### 梯度下降的公式

```
w_new = w_old - 学习率 × 损失函数对w的偏导数
b_new = b_old - 学习率 × 损失函数对b的偏导数
```

用数学符号：

```
w = w - α × ∂L/∂w
b = b - α × ∂L/∂b

其中 α 是学习率（learning rate），控制每一步走多远
```

#### 偏导数怎么算？

回顾一下损失函数：

```
L = Σ(yᵢ - wxᵢ - b)² / n
```

对 w 求偏导：

```
∂L/∂w = -(2/n) × Σxᵢ(yᵢ - wxᵢ - b)
```

对 b 求偏导：

```
∂L/∂b = -(2/n) × Σ(yᵢ - wxᵢ - b)
```

#### 用梯度下降重新算一遍

数据还是上面的 5 个点。

**初始值**：w = 0, b = 0, 学习率 α = 0.0001

**第一轮迭代**：

计算预测值（全是 0）：
```
y_pred = 0×x + 0 = 0
```

计算误差：
```
error₁ = 280 - 0 = 280
error₂ = 340 - 0 = 340
error₃ = 400 - 0 = 400
error₄ = 450 - 0 = 450
error₅ = 520 - 0 = 520
```

计算梯度：
```
∂L/∂w = -(2/5) × (80×280 + 100×340 + 120×400 + 140×450 + 160×520)
       = -(2/5) × (22400 + 34000 + 48000 + 63000 + 83200)
       = -(2/5) × 250600
       = -100240

∂L/∂b = -(2/5) × (280 + 340 + 400 + 450 + 520)
       = -(2/5) × 1990
       = -796
```

更新参数：
```
w = 0 - 0.0001 × (-100240) = 0 - (-10.024) = 10.024
b = 0 - 0.0001 × (-796) = 0 - (-0.0796) = 0.0796
```

哇，w 从 0 直接跳到了 10！太大了！这就是学习率太大的问题。

我们把学习率调小一点，比如 0.00001。

**重新第一轮**：
```
w = 0 - 0.00001 × (-100240) = 1.0024
b = 0 - 0.00001 × (-796) = 0.00796
```

好多了！

**第二轮迭代**... 继续下去，w 和 b 会慢慢接近真实值（2.95 和 44）。

## 📊 第四步：评估模型好坏

### 1. MSE（均方误差）

```python
def mse(y_true, y_pred):
    return np.mean((y_true - y_pred) ** 2)
```

物理意义：误差平方的平均值。单位是"万的平方"。

### 2. RMSE（均方根误差）

```python
def rmse(y_true, y_pred):
    return np.sqrt(mse(y_true, y_pred))
```

物理意义：和原始数据同单位。比如 RMSE = 10，就是平均预测误差 10 万。

### 3. MAE（平均绝对误差）

```python
def mae(y_true, y_pred):
    return np.mean(np.abs(y_true - y_pred))
```

物理意义：平均每次预测误差多少。

### 4. R²（决定系数）

```python
def r2(y_true, y_pred):
    ss_res = np.sum((y_true - y_pred) ** 2)  # 残差平方和
    ss_tot = np.sum((y_true - np.mean(y_true)) ** 2)  # 总平方和
    return 1 - ss_res / ss_tot
```

R² 的含义：
- R² = 1：完美预测
- R² = 0：和用平均值预测一样
- R² < 0：比平均值预测还差
- 越接近 1 越好

## 💻 第五步：代码实现

### 最小二乘法

```python
import numpy as np

def least_squares(x, y):
    """
    最小二乘法求解线性回归

    参数:
        x: 自变量数组，如 [80, 100, 120, 140, 160]
        y: 因变量数组，如 [280, 340, 400, 450, 520]
    返回:
        w: 斜率
        b: 截距
    """
    # 转成 numpy 数组
    x = np.array(x, dtype=float)
    y = np.array(y, dtype=float)

    # 1. 算平均值
    x_mean = np.mean(x)
    y_mean = np.mean(y)

    # 2. 算 w = Σ(x-x̄)(y-ȳ) / Σ(x-x̄)²
    numerator = np.sum((x - x_mean) * (y - y_mean))  # 分子
    denominator = np.sum((x - x_mean) ** 2)             # 分母
    w = numerator / denominator

    # 3. 算 b = ȳ - wx̄
    b = y_mean - w * x_mean

    return w, b


# ===== 测试一下 =====
areas = [80, 100, 120, 140, 160]
prices = [280, 340, 400, 450, 520]

w, b = least_squares(areas, prices)
print(f"拟合结果: 房价 = {w:.2f} × 面积 + {b:.2f}")
# 输出: 房价 = 2.95 × 面积 + 44.00

# 预测 130 平米的房子
pred = w * 130 + b
print(f"130平米的房子预测价格: {pred:.2f} 万")
# 输出: 130平米的房子预测价格: 427.50 万
```

### 梯度下降法

```python
def gradient_descent(x, y, learning_rate=0.00001, epochs=1000):
    """
    梯度下降法求解线性回归

    参数:
        x: 自变量数组
        y: 因变量数组
        learning_rate: 学习率，控制每一步走多远
        epochs: 迭代次数
    返回:
        w, b: 拟合的参数
    """
    x = np.array(x, dtype=float)
    y = np.array(y, dtype=float)

    n = len(x)

    # 1. 初始化参数
    w = 0.0
    b = 0.0

    # 2. 开始迭代
    for epoch in range(epochs):

        # 2.1 计算预测值
        y_pred = w * x + b

        # 2.2 计算误差
        error = y - y_pred  # 真实值减去预测值

        # 2.3 计算梯度
        grad_w = -(2/n) * np.sum(error * x)  # 损失对w的偏导数
        grad_b = -(2/n) * np.sum(error)       # 损失对b的偏导数

        # 2.4 更新参数（往负梯度方向走）
        w = w - learning_rate * grad_w
        b = b - learning_rate * grad_b

        # 2.5 每100轮打印一下进度
        if (epoch + 1) % 200 == 0:
            loss = np.mean(error ** 2)  # MSE
            print(f"Epoch {epoch+1:4d}: w={w:.4f}, b={b:.4f}, loss={loss:.2f}")

    return w, b


# ===== 测试一下 =====
areas = [80, 100, 120, 140, 160]
prices = [280, 340, 400, 450, 520]

print("=== 梯度下降过程 ===")
w, b = gradient_descent(areas, prices, learning_rate=0.00001, epochs=1000)

print(f"\n最终结果: 房价 = {w:.4f} × 面积 + {b:.4f}")
# 输出: 房价 = 2.9500 × 面积 + 44.0000
```

### 用 sklearn（实际项目用这个）

```python
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error, r2_score
import numpy as np

# 1. 准备数据（sklearn 需要 2D 数组）
X = np.array([[80], [100], [120], [140], [160]])
y = np.array([280, 340, 400, 450, 520])

# 2. 创建模型并训练
model = LinearRegression()
model.fit(X, y)

# 3. 查看结果
print(f"斜率 w = {model.coef_[0]:.4f}")     # 2.9500
print(f"截距 b = {model.intercept_:.4f}")    # 44.0000

# 4. 预测
y_pred = model.predict(X)  # 预测训练数据
pred_130 = model.predict([[130]])[0]  # 预测 130 平米
print(f"130平米预测价格: {pred_130:.2f} 万")

# 5. 评估模型
print(f"MSE  = {mean_squared_error(y, y_pred):.2f}")
print(f"RMSE = {np.sqrt(mean_squared_error(y, y_pred)):.2f}")
print(f"R²   = {r2_score(y, y_pred):.4f}")
```

## 🎓 补充：学习率调参技巧

### 什么是学习率？

学习率（learning rate）就是梯度下降时每一步迈多大。

```
lr 太大：步子迈得太大，会"振荡"，甚至发散
lr 太小：步子太小，走得太慢，要很多轮才能到终点
lr 合适：不快不慢，刚刚好
```

### 怎么选学习率？

**经验法则**：
- 从 0.1, 0.01, 0.001, 0.0001 开始试
- 如果 loss 变成 NaN 或者发散了，说明太大
- 如果 loss 下降太慢，可以调大一点

**自适应方法**：
- Adam：自动调整学习率，最常用
- 初始学习率可以大一点，随着训练慢慢减小

## 🎯 线性回归的局限性

### 局限性 1：必须是线性关系

如果数据本身不是线性的，线性回归就拟合不好。

```
非线性数据（线性回归不行）：
    y
    ↑        ╱╲
    │       ╱  ╲
    │      ╱    ╲
    │     ╱      ╲
    └──────────────── x
```

### 局限性 2：容易受离群点影响

离群点（大异常值）会显著影响拟合结果。

### 局限性 3：只能预测连续值

如果要分类（比如判断是猫还是狗），需要用**逻辑回归**等其他算法。
